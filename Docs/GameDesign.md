# 《崇祯模拟器》总设计文档

## 设计定位

《崇祯模拟器》是一款以崇祯元年为开局点的历史政治模拟游戏。玩家扮演新即位的崇祯皇帝，在明末财政、边防、党争、灾荒、内廷与地方治理压力中，通过任免官员、召对问策、批阅奏疏、调度财政与处置事件，尝试延缓、改变或重新解释大明的走向。

游戏追求的是“可推演的历史感”，不是逐条复刻史实。史实用于建立可信的制度、人物和环境底座；玩法允许为了可读性、可操作性和节奏而做清晰的抽象。

## 开局时间

默认开局锚点为：

```text
崇祯元年，公元 1628 年
```

所有环境基建、官职设置、行政区划、NPC 初始状态，默认以这一年作为第一版基准。后续如果引入剧本年份，应明确写入独立 scenario 配置，不直接改动 1628 基准数据。

## 核心原则

游戏管理边界：

```text
皇帝能直接管理/对话的官员：止于副部堂级
皇帝能看见/调度的官员：中央到司局级，地方到州府级
县级及以下：作为事件、舆情、税粮、地方叙事和 LLM 语料来源
```

这意味着玩家不会直接管理每个县令、胥吏、士绅、军户或商人，但这些群体会通过奏疏、灾情、案件、地方压力和人物关系进入游戏。

## 技术与美术基线

- 平台：iOS SwiftUI。
- 当前主界面：群星谱，展示 NPC 档案、天罡、命数、心盘、势网。
- 字体：`LXGWWenKai-Medium.ttf`，统一作为游戏中文字体。
- 文本 LLM：DeepSeek 路线，用于开发期文学/数值支持和游戏内对白、奏疏、事件文本。
- 图像 LLM：Google Gemini 图像路线，用于开发期美术资产和未来游戏内插画、图标、反馈图。
- API Key 不进入仓库；开发期通过本地环境变量或 Xcode Scheme 注入。

## 数据基建分层

项目当前把数据分成三类：

```text
Tiangang seed
NPCDatabase
EnvironmentDatabase
```

`Tiangang seed` 是 36 维人物坐标的维度字典。

`NPCDatabase` 是人物基建，当前覆盖 263 名 NPC，包括：

- 核心身份
- 开局位置
- 官职品秩
- 任官记录
- 状态时间线
- 关系边
- 势网
- 天罡 36 维
- 心盘
- 命数
- 外貌画像基因
- 美术资产
- 小传
- 命批

`EnvironmentDatabase` 是环境基建，当前包括：

- 正式行政区划
- 既有地点索引
- 机构
- 官职
- 品秩
- 地点别名
- 宦官机构
- 宦官服饰标记

## 正式行政区划

第一版正式行政区划锚定崇祯元年，做到府/直隶州一级，不展开精确县级行政表。

层级：

```text
大明
  ├─ 北直隶
  ├─ 南直隶
  ├─ 山东
  ├─ 山西
  ├─ 河南
  ├─ 陕西
  ├─ 四川
  ├─ 江西
  ├─ 湖广
  ├─ 浙江
  ├─ 福建
  ├─ 广东
  ├─ 广西
  ├─ 云南
  └─ 贵州
      └─ 府 / 直隶州 / 军民府 / 西南土司性质府州
```

当前数据规模：

```text
根节点：1
一级区：15
府级节点：203
```

府级节点拆分：

```text
普通府：145
直隶州：29
军民府：13
西南土司/羁縻性质府州：16
```

每个府级节点附带 `subordinate_places_text`，作为连续文本地名语料。该字段用于剧情、奏疏、事件落点和 LLM 生成，不作为精确县级行政区划、赋税口径或司法辖区依据。

## 地方常设官职

地方官制采用游戏化简化，不追求完全复刻明制品级。第一版地方常设主官：

```text
从二品：布政使      民政财政
正三品：按察使      司法监察
从三品：都指挥使    军事卫所
正四品：州府主官    知府 / 知州 / 军民府主官 / 土府州主官
```

按照当前行政区划：

```text
15 个一级区 × 3 个省级主官 = 45
203 个府级节点 × 1 个州府主官 = 203
地方常设主官合计 = 248
```

在未来玩法里，布政使、按察使、都指挥使分别承担省级财政民政、司法监察、军事卫所的数值入口；州府主官承担地方执行和事件落点。

## 中央组织架构

中央第一版按玩家可读性简化为：

```text
皇帝
├─ 内阁
├─ 六部
├─ 都察院
├─ 通政司
├─ 大理寺
└─ 内廷
```

六部固定为：

```text
吏部：官员任免、考核、升降、党争人事
户部：财政、税赋、粮草、赈灾、国库
礼部：礼制、科举、外交、祭祀、名分
兵部：军令、边防、调兵、武官任免
刑部：案件、律法、审判
工部：水利、城防、工程、制造
```

六部单部简化编制：

```text
尚书：1
左侍郎：1
右侍郎：1
郎中：4
```

六部合计：

```text
每部 7 人 × 6 = 42 人
```

中央官职原则：

```text
尚书、侍郎等副部堂以上：皇帝可直接召对
郎中等司局级：皇帝可见，可调度、查办、派差，但通常不直接日常对话
```

总督、巡抚属于特别授权或危机系统职位，暂不作为普通常设三司结构的一部分。后续可按剧本、战区、财政或灾荒事件动态设立。

## NPC 数值系统

NPC 是游戏政治推演的核心资产。当前主要数值层：

### 天罡

36 维人物坐标，分政治立场与专业能力。政治维度用于判断价值倾向、站队、政策偏好；专业维度用于判断实际办事能力。

### 心盘

心盘是人物面对皇帝与局势时的动态心理入口。当前初始字段包括：

```text
道合
势合
畏惧
信任系数
仇恨
象限
核心关切
```

未来运行时，心盘应随赏罚、召对、党争、失败、流言和事件后果变化。

### 命数

命数是人物性情和行为模式的中层抽象。当前为 10 轴，用于解释一个人遇压、冒险、守名、信任、结群、记怨等倾向。

### 势网

势网记录 NPC 的关系资本、派系嵌入、恩庇网络、敌对压力和关键关系。它是党争、举荐、弹劾、结盟、背锅和救援的重要基础。

## 环境数值系统待建

行政区划和官僚架构只是环境基建第一层。后续需要补运行时环境变量：

- 国库
- 粮储
- 税赋压力
- 民心
- 灾荒
- 流民
- 治安
- 边防压力
- 官场腐败
- 朝堂派系温度
- 内廷干政强度
- 奏疏拥堵

这些变量应挂在不同层级：

```text
全国级：国库、总体民心、朝堂温度、边防总压
一级区：省级财政、灾荒、治安、赋税、兵备
府州级：地方事件、民变风险、粮价、案件、官声
```

## LLM 使用原则

LLM 不直接决定核心数值真相。它主要用于：

- 生成奏疏、对白、事件叙事、传记和命批
- 辅助提出数值方案
- 为美术图像生成 prompt
- 作为数据校验的提醒源

LLM 输出必须进入人工或脚本校验流程。历史事实争议不能只听一个模型；DeepSeek 可以提醒，必要时再用 Google/Web 或其它资料交叉复核。

## 当前已确认事项

- 游戏开局时间：崇祯元年，1628 年。
- 正式行政区划做到府/直隶州一级。
- 州府同级，州府主官统一正四品。
- 地方一级区使用北直隶、南直隶、十三布政司。
- 地方常设主官按 15 个一级区和 203 个府级节点计算，共 248 人。
- 皇帝直接管理止于副部堂级。
- 皇帝可见范围：中央到司局级，地方到州府级。
- 县级及以下作为语料、事件和环境压力来源，不作为常设可管理官位。

## 待决事项

- 内阁人数与阁臣品级如何游戏化。
- 都察院、通政司、大理寺、内廷的常设编制人数。
- 总督、巡抚是否作为事件职位、危机职位或可任命特别职位。
- 环境运行时变量的具体字段和初始值。
- 州府主官与 NPCDatabase 的任命映射方式。
- 地方财政、灾荒、治安与人物能力的结算规则。
