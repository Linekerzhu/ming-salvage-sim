# 官僚班子执行评估：方案审计与落地

## 问题

组织图已经能显示朝廷机构、席位、空缺和在任者，但这些信息过去主要是展示层。月末政令推演会读取承办人画像、天罡摘要、心盘、召对立场和派系盘面，却缺少一层可审计的“班子是否配齐、承办链是否匹配”的中间判断。

玩家因此容易感觉：任命强臣、补齐官署、一岗多配，究竟怎样影响政令推行，不够直观。

## 方案比选

### 方案 A：继续纯 LLM 推演

- 优点：改动小，叙事自由。
- 缺点：空额、一岗多配、官署匹配度不稳定；玩家很难追溯为什么某道旨意折损。
- 结论：保留作为叙事层，但不能独立承担机制解释。

### 方案 B：全公式硬结算

- 优点：最可预测，最容易做数值平衡。
- 缺点：需要重写月末结算，且会把晚明政治压成单一公式；钱粮、地方、派系、人物心态很难在公式里保持细腻。
- 结论：暂不采用，适合后续做高风险重构时再评估。

### 方案 C：确定性预评估 + LLM 叙事落地

- 优点：先用代码把班子完整度、关键空缺、超配、承办人适配、心盘、派系和召对背书压成结构化证据；再由月末推演把证据转成具体奏章下落。
- 缺点：不是最终硬结算，仍依赖推演官按提示正确使用证据。
- 结论：采用。它能以较小改动把“官僚体系影响政令”前台化，并为后续硬公式保留接口。

## 当前落地

新增 `ming_sim/bureaucracy.py`：

- `base_institution_specs()`：统一基础机构与席位定义，供组织图和执行评估复用。
- `organization_diagnostics()`：计算整体朝廷执行力、每个机构执行力、席位覆盖、在任质量、关键空缺、一岗多配和兼差拥挤。
- `directive_execution_assessments()`：为每道旨意计算执行预评估，输出 `score/label/domains/relevant_institutions/drivers/risks/execution_hint`。

接入点：

- `web_app.py` 的组织图 API 合并机构执行力与风险摘要。
- `web/src/main.tsx` 与 `web/src/styles.css` 展示整体和机构级执行力。
- `ming_sim/simulation.py` 把 `bureaucracy_brief` 与 `execution_assessments` 注入月末推演 payload。
- `content/prompts/season_simulator.md` 明确要求推演官使用执行预评估，低分旨意不得写成无阻顺行。

## 机制口径

执行评估不是“自动成败判定”，而是“政令预审”：

- 高分表示班子、承办人、心盘、派系与召对背书基本相合。
- 中分表示可以启动，但需要写明折损环节。
- 低分表示应写出具体阻力，除非诏书另给补缺、授权、钱粮或强力抓手。

这让“配齐班子”和“点对承办人”成为玩家可见、可推理、可复盘的政治操作。
